If TransactionData is included:

Aggregate MeterValues from multiple TransactionData in one big, happy list. TransactionData is just a container for MeterValues without any additional data/semantics anyway. This enables us to write into DB in one repository call (query), rather than multiple calls as it was before (for each TransactionData)

Saved the world again with this micro-optimization.
