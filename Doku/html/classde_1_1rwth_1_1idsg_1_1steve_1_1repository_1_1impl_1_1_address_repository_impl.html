The call site does not care about the database internal specifics, but only wants to save the address. We return

  • null, if address was empty (no field set)
  • a new address_pk, if we have to do an INSERT
  • address_pk from the input parameter (for convenience)

to spare us the decision logic in the call site, so that it can just reference the returned value in parent's table.
